Protocolos de Red

Protocolos Definición
TCP (Transmission Control Protocol) Es un protocolo de comunicación orientado a la conexión que garantiza la entrega fiable de datos entre dos dispositivos en una red. TCP asegura que los paquetes de datos lleguen en orden y sin errores, corrigiendo posibles fallos en la transmisión. Es uno de los protocolos más utilizados en la suite de protocolos de Internet (TCP/IP).
HTTP (HyperText Transfer Protocol) Es un protocolo de aplicación utilizado para transferir información en la web. Es el protocolo básico que los navegadores y servidores web utilizan para comunicarse. HTTP es un protocolo sin estado, lo que significa que no guarda información entre diferentes solicitudes (a menos que se utilicen otras tecnologías como cookies).
HTTPS (HyperText Transfer Protocol Secure) Es una versión segura de HTTP que utiliza SSL (Secure Socket Layer) o TLS (Transport Layer Security) para cifrar la comunicación entre el navegador y el servidor web. HTTPS protege la confidencialidad e integridad de los datos transmitidos y se utiliza principalmente en sitios que manejan información sensible, como contraseñas o información bancaria.
FTP (File Transfer Protocol): Es un protocolo de red que se utiliza para transferir archivos entre un cliente y un servidor. FTP permite cargar o descargar archivos, y también realizar otras operaciones con archivos, como renombrarlos o eliminarlos. Sin embargo, FTP en su forma básica no cifra los datos, por lo que puede ser inseguro para transmitir información sensible a través de redes no confiables.
TCP/IP (Transmission Control Protocol/Internet Protocol): Es la suite de protocolos fundamentales que permiten la comunicación en Internet. TCP/IP combina el TCP, que maneja la transmisión fiable de datos, con IP (Internet Protocol), que se encarga de enrutar y entregar los paquetes de datos a través de las redes. El modelo TCP/IP es la base sobre la cual se construyen muchos otros protocolos, como HTTP, HTTPS, FTP, y más.
